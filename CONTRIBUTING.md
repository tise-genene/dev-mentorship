# Contributing Guidelines

Welcome to the Eagle Dev Mentorship repo!  
Follow these steps when contributing:

1. **Fork** this repository.
2. **Clone** your fork locally.
3. Create a **new branch** for each feature:  
   `git checkout -b feature/<feature-name>`
4. Commit frequently and write clear messages.
5. Push your branch and open a **Pull Request**.
6. Reference the issue number in your PR description (e.g., `Fixes #12`).

Please ensure your code:

- Runs without errors
- Uses consistent naming conventions
- Has clear, minimal comments
- Is reviewed before merge
