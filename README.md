# Eagle Dev Mentorship 🦅

This repository is a mentorship playground for beginner developers to practice real-world collaboration using GitHub issues and pull requests.

## 🚀 How It Works
1. Pick an open issue from the list.
2. Fork the repository.
3. Create a new branch (e.g., `feature/weather-app`).
4. Implement the feature.
5. Submit a Pull Request referencing the issue (e.g., `Fixes #3`).
6. Wait for code review and feedback.

## 🧩 Rules
- PRs must be clean, formatted, and well-commented.
- Use meaningful commit messages.
- Never merge your own PRs — all changes go through review.
- Keep learning, keep improving.
